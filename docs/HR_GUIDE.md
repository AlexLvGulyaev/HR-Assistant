# Руководство HR-специалиста

Это руководство для HR-специалистов, работающих с результатами обработки резюме в HR Assistant.

---

## Роли в системе

| Роль | Где работает | Назначение |
|------|-------------|------------|
| **Кандидат** | Telegram | Отправляет резюме |
| **HR-специалист** | База данных | Анализирует результаты matching |
| **Администратор** | Admin Console | Мониторит систему |

HR-специалист работает с результатами обработки в базе данных или через интеграцию с HR-системой.

---

## Как работает система

### Автоматический процесс

1. Кандидат отправляет резюме через Telegram (текст, голос, документ, изображение)
2. Система извлекает структурированные данные (ФИО, контакты, опыт, навыки)
3. Система выполняет matching с открытыми вакансиями
4. Система формирует score (0-100) и recommendation (match/no_match)
5. Кандидат получает результат matching
6. Данные сохраняются в базе данных

### Результаты для HR-специалиста

HR-специалист получает:
- Структурированный профиль кандидата
- Результат matching с score
- Обоснование решения (reason)
- Контакты кандидата

---

## Работа с результатами matching

### Понимание Score

**Score (0-100):** оценка соответствия кандидата вакансии.

| Score | Уровень | Что означает |
|-------|--------|--------------|
| **80-100** | Высокий | Кандидат полностью соответствует требованиям |
| **60-80** | Средний | Кандидат частично соответствует, возможны компромиссы |
| **40-60** | Низкий | Кандидат слабо соответствует, требуется обучение |
| **< 40** | Несоответствие | Кандидат не соответствует требованиям |

---

### Понимание Decision

**Decision:** рекомендация системы.

| Decision | Что означает |
|----------|--------------|
| **match** | Кандидат подходит для вакансии |
| **no_match** | Кандидат не подходит |

## Работа с извлечёнными данными

### Структура данных кандидата

Система автоматически извлекает и хранит данные в двух таблицах:

**Таблица `candidates` — профиль кандидата:**

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | UUID | Уникальный идентификатор |
| `full_name` | TEXT | ФИО кандидата |
| `city` | TEXT | Город проживания |
| `desired_position` | TEXT | Желаемая должность |
| `experience_years` | NUMERIC | Опыт работы (в годах) |
| `skills` | TEXT[] | Навыки (массив) |
| `salary_expectation` | NUMERIC | Зарплатные ожидания |
| `candidate_summary` | TEXT | Краткое описание |
| `data_quality_status` | TEXT | Статус качества данных |

**Таблица `candidate_contacts` — контакты кандидата:**

| Поле | Тип | Описание |
|------|-----|----------|
| `candidate_id` | UUID | Ссылка на кандидата |
| `contact_type` | TEXT | Тип контакта (email, phone) |
| `contact_value` | TEXT | Значение контакта |
| `normalized_value` | TEXT | Нормализованное значение |
| `is_primary` | BOOLEAN | Основной контакт |

**Примечание:** Контакты извлекаются автоматически и хранятся отдельно от профиля кандидата.

---

### Качество извлечения

**Текстовое резюме:** Высокое качество (данные извлечены корректно)

**Голосовое резюме:** Среднее качество (возможны неточности из-за STT)

**Документ (PDF/DOCX):** Высокое качество (при корректном форматировании)

**Изображение:** Среднее/низкое качество (зависит от качества фото и OCR)

**Рекомендация:** При среднем/низком качестве данных — запросить у кандидата подтверждение.

---

### Проверка данных

HR-специалист может:

1. **Проверить извлечённые данные** — сравнить с исходным резюме
2. **Дополнить данные** — добавить недостающие поля
3. **Исправить ошибки** — скорректировать неточности

---

### Структура данных matching

**Таблица `matches` — результаты сопоставления:**

| Поле | Тип | Описание |
|------|-----|----------|
| `candidate_id` | UUID | Ссылка на кандидата |
| `vacancy_id` | UUID | Ссылка на вакансию |
| `score` | NUMERIC | Итоговая оценка (0-100) |
| `decision` | TEXT | Решение (match/no_match) |
| `reason` | TEXT | Обоснование решения |

**Детальные оценки в `raw_llm_response`:**

| Поле | Баллы | Описание |
|------|-------|----------|
| `role_score` | 0-30 | Соответствие должности/роли |
| `skills_score` | 0-35 | Соответствие навыков |
| `experience_score` | 0-20 | Соответствие опыта |
| `conditions_score` | 0-15 | Город, формат, зарплата |

**Правила принятия решения:**
- `score >= 60` → `decision = "match"`
- `score < 60` → `decision = "no_match"`

## Ограничения системы

### 1. Ограничения извлечения данных

**Не извлекается автоматически:**
- Детальное описание опыта работы (компании, проекты)
- Образование
- Сертификаты
- Языки

**Требуется ручной ввод:**
- Дополнительные навыки
- Предпочтения по формату работы (офис/удалёнка)
- Гражданство

---

### 2. Ограничения matching

**Не учитывается:**
- Soft skills (коммуникация, лидерство)
- Cultural fit
- Конкретные проекты
- Образование

**Учитывается:**
- Опыт работы (в годах)
- Навыки (hard skills)
- Город
- Зарплатные ожидания

---

### 3. Ограничения мультимодальности

**Голосовые сообщения:**
- Качество зависит от STT-сервиса
- Возможны ошибки транскрибации

**Изображения:**
- Качество зависит от OCR
- Возможны ошибки распознавания

**Документы:**
- Поддерживаются только PDF и DOCX
- Форматирование может теряться

## FAQ HR-специалиста

### Как понять, что данные извлечены корректно?

Проверьте extracted fields с исходным резюме. Если есть неточности — запросите у кандидата подтверждение.

---

### Что делать, если score низкий, но кандидат интересен?

Score — рекомендация, не обязательство. HR-специалист может:
- Назначить интервью
- Запросить дополнительные данные
- Предложить другую вакансию

---

### Как связаться с кандидатом?

Используйте контакты из таблицы `candidate_contacts`:

```sql
SELECT contact_type, contact_value, normalized_value
FROM candidate_contacts
WHERE candidate_id = '<candidate_id>'
ORDER BY is_primary DESC;
```

**Типы контактов:**
- `email` — email кандидата
- `phone` — телефон кандидата

---

### Что делать, если контакты не извлечены?

Если контакты отсутствуют:
1. Кандидат получил ответ от бота
2. Кандидат ожидает звонка
3. Попросите кандидата отправить контакты повторно

---

### Как добавить вакансию в систему?

Вакансии добавляются администратором в базу данных:

```sql
INSERT INTO vacancies (title, description, requirements, salary_min, salary_max, status)
VALUES (
  'Название вакансии',
  'Описание вакансии',
  'Требования к кандидату',
  150000,  -- salary_min
  220000,  -- salary_max
  'open'   -- status: 'open' или 'closed'
);
```

**Обязательные поля:**
- `title` — название вакансии
- `description` — описание
- `status` — статус ('open', 'closed')

**Опциональные поля:**
- `requirements` — требования
- `salary_min`, `salary_max` — вилка зарплат

---

### Как посмотреть историю кандидата?

История хранится в таблицах:
- `candidates` — профиль кандидата
- `matches` — результаты matching
- `final_decisions` — итоговые решения

---

### Что делать при ошибке обработки?

Если система не смогла обработать резюме:
1. Кандидат получил fallback-сообщение
2. Попросите кандидата отправить резюме повторно
3. Или обработайте вручную

## Рекомендации по работе

### 1. Приоритизация кандидатов

**Высокий приоритет (score 80+):**
- Связаться в течение 24 часов
- Назначить интервью

**Средний приоритет (score 60-80):**
- Рассмотреть в течение 3 дней
- Оценить дополнительные факторы

**Низкий приоритет (score < 60):**
- Рассмотреть для других вакансий
- Или отклонить

---

### 2. Валидация данных

**Обязательно проверяйте:**
- Контакты (email, телефон)
- Опыт работы (годы)
- Навыки (ключевые технологии)

**При сомнениях:**
- Запросите подтверждающие документы
- Назначьте звонок-знакомство

---

### 3. Коммуникация с кандидатами

**Используйте извлечённые данные:**
- Обращайтесь по имени (`full_name`)
- Упоминайте вакансию (`vacancy.title`)
- Объясняйте решение (`match.reason`)
