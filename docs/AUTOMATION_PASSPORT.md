## Назначение

**Бизнес-цель:** Автоматизация первичной обработки резюме и подбора вакансий.

**Решаемая проблема:**
- Медленная обработка резюме (10-15 минут на резюме)
- Ручное извлечение данных из резюме
- Отсутствие автоматизированного matching кандидатов с вакансиями
- Ограниченные форматы ввода резюме

**Целевая аудитория:**
- HR-специалисты
- Рекрутеры
- Кандидаты (через Telegram-бот)

## Диагностика и устранение неполадок

### Типовые инциденты

#### Инцидент 1: Обработка зависла

**Симптомы:**
- Записи в статусе `processing_started` > 5 минут
- Кандидаты не получают ответ

**Диагностика:**
```sql
SELECT COUNT(*) FROM candidate_inputs
WHERE processing_status = 'processing_started'
  AND created_at < NOW() - INTERVAL '5 minutes';
```

**Решение:**
1. Проверить Watchdog workflow
2. Сбросить зависшие записи вручную:
```sql
UPDATE candidate_inputs
SET processing_status = 'prepared'
WHERE processing_status = 'processing_started'
  AND created_at < NOW() - INTERVAL '5 minutes';
```

---

#### Инцидент 2: Telegram Bot недоступен

**Симптомы:**
- Webhook не работает
- Кандидаты не могут отправить резюме

**Диагностика:**
```bash
curl -X GET "https://api.telegram.org/bot${TELEGRAM_BOT_TOKEN}/getWebhookInfo"
```

**Решение:**
1. Проверить WEBHOOK_URL
2. Переустановить webhook
3. Проверить SSL сертификат

---

#### Инцидент 3: OpenAI API недоступен

**Симптомы:**
- Ошибки извлечения данных
- Ошибки matching

**Диагностика:**
- Проверить https://status.openai.com
- Проверить логи n8n (узлы OpenAI)

**Решение:**
1. Ждать восстановления API
2. Увеличить retry attempts
3. Сообщить пользователям о задержке

## Процедуры управления жизненным циклом

### Развёртывание

**Процедура:**
1. Создать бэкап БД
2. Остановить n8n workflows
3. Применить миграции БД
4. Обновить workflows
5. Запустить n8n workflows
6. Проверить healthcheck

**Откат:**
1. Восстановить БД из бэкапа
2. Откатить workflows к предыдущей версии
3. Перезапустить n8n

---

### Обновление

**Частота:**
- Security updates: немедленно
- Feature updates: по расписанию (раз в месяц)
- Dependency updates: по необходимости

**Процедура:**
1. Тестирование в staging
2. Code review
3. Деплой в production
4. Smoke tests
5. Мониторинг

---

### Удаление

**Процедура вывода из эксплуатации:**
1. Уведомить пользователей за 30 дней
2. Остановить приём новых резюме
3. Экспортировать данные
4. Удалить workflows
5. Удалить БД
6. Удалить инфраструктуру
