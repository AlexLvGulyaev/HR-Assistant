## Анализ по сегментам

**Характеристика:** Кандидаты с точным совпадением должности и навыков.

**Результаты:**

- Ухудшение: **+46.3%**

**Вывод:** Prompt B работает **хуже** на очевидных совпадениях. Увеличивает ошибку почти в полтора раза.

---

### obvious_no_match

**Характеристика:** Кандидаты с полным несоответствием должности и навыков.

- MAE_B: **15.57** (хуже)

**Вывод:** Prompt B работает **хуже** на очевидных несоответствиях, но обе модели корректно определяют `no_match` в 100% случаев.

---

### borderline

**Характеристика:** Кандидаты с частичным совпадением, пограничные случаи.

**Вывод:** Prompt B показывает **наихудшие результаты** именно на пограничных случаях — там, где детальные инструкции должны были помочь. Это ключевой провал гипотезы.

## Паттерны ухудшения

### 1. Завышение score на borderline кейсах

В кейсах с reference_score около 30-50, Prompt B склонен:
- Завышать score до 60-88
- Неправильно классифицировать как `match`
- Давать избыточно оптимистичные оценки

### 2. Занижение score на borderline кейсах

В других borderline кейсах, Prompt B склонен:
- Занижать score до 0-20
- Давать избыточно пессимистичные оценки
- Переоценивать несоответствия

### 3. Нестабильность на obvious_match

Даже на очевидных совпадениях (`HRA-EVAL-000006`, `HRA-EVAL-000002`) Prompt B может:
- Выдать score = 0
- Полностью игнорировать совпадения

**Вопрос:** Показывает ли Prompt B преимущество хотя бы на одной из категорий данных?

**Ответ:** **Нет.**

Prompt B уступает Prompt A на всех трёх сегментах:

**Гипотезы:**
- Избыточная детализация создаёт информационный шум

**Идеи:**
