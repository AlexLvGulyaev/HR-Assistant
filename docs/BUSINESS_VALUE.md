# Ценность для бизнеса

Документ объясняет, какую бизнес-ценность приносит HR Assistant и какие проблемы решает.

---

## Что такое HR Assistant

**HR Assistant** — мультимодальный AI-ассистент для автоматизации первичной обработки резюме и подбора вакансий. Система принимает резюме в любом формате через Telegram-бота, автоматически извлекает структурированные данные кандидата, сравнивает профиль с открытыми вакансиями и формирует мультимедийный ответ.

Кандидат отправляет резюме в удобном формате (текст, голос, документ, изображение), система анализирует за секунды и возвращает результат matching с вакансиями.

---

## Какие проблемы бизнеса решает система

### Медленная обработка резюме

**Проблема:** HR-специалист тратит 10-15 минут на первичный анализ каждого резюме: чтение, извлечение ключевых данных, оценка соответствия вакансии.

**Решение:** Автоматическое извлечение данных и matching за < 1 минуту. Система анализирует резюме и сравнивает с вакансиями без участия HR.

**Эффект:** Сокращение времени на первичный отсив в 10-15 раз.

---

### Ограниченные форматы ввода

**Проблема:** Кандидаты отправляют резюме в разных форматах: PDF, DOCX, фото, голосовые сообщения. HR вынужден конвертировать, распечатывать, прослушивать.

**Решение:** Мультимодальный ввод — система принимает текст, голос, документы, изображения и автоматически приводит к единому формату.

**Эффект:** Снижение барьера для кандидатов, повышение конверсии откликов.

---

### Ручное извлечение данных

**Проблема:** HR вручную переносит данные из резюме в CRM: ФИО, контакты, опыт, навыки, зарплатные ожидания. Риск ошибок и пропусков.

**Решение:** Автоматическое извлечение структурированных данных с помощью LLM (GPT-4o-mini). ФИО, город, должность, опыт, навыки, зарплата — автоматически.

**Эффект:** Исключение рутинной работы, унифицированный формат данных.

---

### Отсутствие сравнения с вакансиями

**Проблема:** HR вручную сравнивает профиль кандидата с каждой вакансией. Субъективная оценка, нет стандартизированного scoring.

**Решение:** Автоматический matching — система сравнивает профиль кандидата с открытыми вакансиями, формирует score (0-100) и recommendation (match/no_match) на основе детальной системы оценки.

**Эффект:** Консистентная оценка, сокращение времени на анализ.

---

### Нет мультимедийного ответа

**Проблема:** Кандидат получает только текстовый ответ. Нет персонализированного визуального представления результатов.

**Решение:** Мультимедийный вывод — текст + голос (TTS) + визуальные материалы (инфографика). Кандидат получает информацию в удобном формате.

**Эффект:** Повышение вовлечённости кандидатов, лучший UX.

---

## Что получает бизнес после внедрения

### Автоматизация обработки

Резюме в любом формате автоматически принимаются, анализируются и сравниваются с вакансиями без участия HR.

**Форматы ввода:**
- Текстовое сообщение
- Голосовое сообщение
- Документ (PDF, DOCX)
- Изображение (фото резюме)

**Извлекаемые данные:**

| Поле | Тип | Описание |
|------|-----|----------|
| `full_name` | string | ФИО кандидата |
| `city` | string | Город проживания |
| `desired_position` | string | Желаемая должность |
| `experience_years` | number | Опыт работы (в годах) |
| `skills` | array | Массив навыков |
| `salary_expectation` | number | Зарплатные ожидания |
| `summary` | string | Краткое описание профиля |

**Контакты (хранятся отдельно):**

| Поле | Тип | Описание |
|------|-----|----------|
| `email` | string | Email кандидата |
| `phone` | string | Телефон кандидата |

---

### Быстрая обработка

| Время | Операция |
|-------|----------|
| < 30 сек | Текстовый ответ |
| < 60 сек | Ответ с TTS (голос) |
| < 90 сек | Ответ с визуальными материалами |
| < 180 сек | Ответ с видео |

**Сравнение:**
| До | После |
|----|-------|
| 10-15 минут на анализ | < 1 минута на обработку |
| Ручное извлечение данных | Автоматическое извлечение |
| Субъективная оценка | Консистентный scoring |

---

### Структурированные данные

Каждое резюме преобразуется в унифицированную структуру:

**Профиль кандидата:**
```json
{
  "full_name": "Иванов Иван Иванович",
  "city": "Москва",
  "desired_position": "Frontend Developer",
  "experience_years": 5,
  "skills": ["React", "TypeScript", "Node.js"],
  "salary_expectation": 180000,
  "summary": "Frontend-разработчик с 5-летним опытом..."
}
```

**Контакты кандидата (отдельная таблица):**
```json
[
  {
    "contact_type": "email",
    "contact_value": "ivanov@example.com",
    "is_primary": true
  },
  {
    "contact_type": "phone",
    "contact_value": "+79001234567",
    "is_primary": true
  }
]
```

---

### Автоматический matching

Система сравнивает профиль кандидата с открытыми вакансиями по детальной системе оценки:

**Система оценки (итого 100 баллов):**

| Критерий | Баллы | Описание |
|----------|-------|----------|
| **Должность / роль** | 0-30 | Соответствие желаемой должности и роли вакансии |
| **Навыки** | 0-35 | Пересечение навыков кандидата с требованиями вакансии |
| **Опыт** | 0-20 | Соответствие опыта работы требованиям |
| **Город / формат / зарплата** | 0-15 | Город, формат работы, зарплатные ожидания |
| **Итого** | **0-100** | Максимальный score |

**Порог для match:** score >= 60

**Результат matching:**
```json
{
  "vacancy_id": "vac-001",
  "title": "Senior Frontend Developer",
  "role_score": 25,
  "skills_score": 30,
  "experience_score": 20,
  "conditions_score": 10,
  "score": 85,
  "decision": "match",
  "reason": "Кандидат имеет 5 лет опыта (требуется 5+), владеет React и TypeScript (ключевые навыки), город Москва совпадает, зарплатные ожидания в диапазоне."
}
```

**Примеры matching:**

| Score | Decision | Обоснование |
|-------|----------|-------------|
| 85 | match | Полное соответствие опыта, навыков, города, зарплаты |
| 65 | match | Частичное соответствие, но выше порога (60) |
| 45 | no_match | Недостаточное соответствие, ниже порога |

---

### Мультимедийный ответ

Кандидат получает результат в нескольких форматах:

**Текст:** краткая информация о matching
**Голос (TTS):** голосовое сообщение с результатами
**Визуальные материалы:** инфографика с профилем кандидата

## Ключевые возможности системы

### Мультимодальный ввод

Приём резюме в любом формате: текст, голос, документ, изображение. Автоматическое преобразование к единому формату.

### AI-извлечение данных

Автоматическое извлечение структурированных данных с помощью GPT-4o-mini. ФИО, контакты, опыт, навыки — без ручного ввода.

### Matching с вакансиями

Детальное сравнение профиля кандидата с открытыми вакансиями:
- Должность / роль: 0-30 баллов
- Навыки: 0-35 баллов
- Опыт: 0-20 баллов
- Город / формат / зарплата: 0-15 баллов

Scoring (0-100) и recommendation (match/no_match).

### Мультимедийный вывод

Генерация ответа в нескольких форматах: текст, голос (TTS), изображения (инфографика), видео.

### Telegram Bot

Всё взаимодействие через Telegram-бота. Удобный интерфейс для кандидатов.

### Сокращение времени обработки

| Метрика | До | После |
|---------|----|-------|
| Время на анализ резюме | 10-15 минут | < 1 минута |
| Извлечение данных | Вручную | Автоматически |

**Рост производительности:** 10-15x

---

### Повышение конверсии

- **Мультимодальный ввод** — кандидаты могут отправить резюме в удобном формате
- **Быстрый ответ** — кандидаты получают результат за минуты, не дни
- **Персонализированный UX** — голос и визуальные материалы повышают вовлечённость

---

### Унификация данных

- Все резюме в едином структурированном формате
- Консистентный scoring для всех кандидатов (детальная система оценки)
- Возможность интеграции с HR-системами (ATS, CRM)

---

### Снижение рутинной работы

HR-специалист освобождается от:
- Ручного чтения резюме
- Извлечения данных
- Заполнения CRM
- Первичного matching

## Почему HR Assistant отличается от ручной обработки

**HR-специалист получает не сырые резюме, а уже структурированные данные:**

- ✅ Профиль кандидата (ФИО, контакты, опыт, навыки)
- ✅ Результат matching (детальный score по критериям, decision, reason)
- ✅ Готовое резюме в унифицированном формате
- ✅ Рекомендация по вакансии

| Аспект | Ручная обработка | HR Assistant |
|--------|------------------|--------------|
| **Время анализа** | 10-15 минут | < 1 минута |
| **Форматы резюме** | Ограничены | Мультимодальный ввод |
| **Извлечение данных** | Вручную | Автоматически (LLM) |
| **Matching** | Субъективный | Детальный scoring (30+35+20+15) |
| **Формат ответа** | Только текст | Текст + голос + визуал |
